# Agent辅助证明Delaunay三角剖分Stretch Factor上界：宽松验证型系统架构设计

## 1 问题背景

### 1.1 数学问题

Delaunay三角剖分的stretch factor上界是计算几何中的长期开放问题。Xia (2013) 通过引入链（chain）与势函数分析，证明了上界小于 1.998。其证明主线是构造目标函数

$$
\Upsilon_O(u,v)=|P_O(u,v)|-\lambda |D_O(u,v)|+\Phi_O,
$$

并证明：

- **Lemma 2**：对所有链 \(O\) 与终点 \(u,v\)，都有 \($\Upsilon_O(u,v)<0$\)
- **Lemma 3**：对某个极端链 \(O^\*\)，有 \($\Phi_{O^*}\ge -C\cdot |P_{O^*}(u,v)|$\)

于是得到最终上界

$$
\rho=\frac{\lambda}{1-C}.
$$

因此，改进上界的核心仍然是：**设计更好的势函数 $\Phi_O$，并用可验证的方式同时控制 Lemma 2 与 Lemma 3 中出现的关键常数。**

---

### 1.2 原证明的真正瓶颈

原证明中，很多结构性结论其实是为了让分析“可做”，而不是势函数本身必须满足的唯一形式。尤其有两个地方最容易限制搜索空间：

#### （1）终点极值位置被压到端点或pivot点

在 Section 4 中，Xia 先证明当 \($\Phi_O$\) 不依赖 \(v\) 时，\($\Upsilon_O(u,v)$\) 关于终点位置可以被写成“常数 + 线性项 + 凸项”，于是最大值只可能出现在 unobstructed arc 的端点或 pivotal 点。这个结论非常强，但它实际上要求势函数在终点变量上有较强的结构限制。

#### （2）多变量导数条件被压成四个单变量函数

在 Proposition 7 中，作者需要控制变形过程里

$$
\frac{\partial \Upsilon_O(u,v)}{\partial X_{o_n}}
$$

的符号。原证明通过一系列边界化简，把问题压缩成四个单变量函数 \($g_1(\alpha),g_2(\alpha),g_3(\alpha),g_4(\alpha)\lt 0$\)，最后再用 Lipschitz 常数和 Piyavskii 算法验证。

这条路线非常漂亮，但也意味着：

- 新势函数若引入 \(v\)-dependence，可能直接无法复用端点/pivot极值结论；
- 新势函数若引入多参数耦合项，往往无法再被压成单变量函数；
- 很多潜在更强的候选，会在“分析形式不匹配”而不是“数学上不成立”时被过早剪枝。

---

### 1.3 宽松化的基本原则

新的agent不应放松正确性，而应放松“证书的形状”。

也就是说：

- **保持真正结构性的硬约束不变**
- **把端点极值、单变量验证等从硬编码结论改成可选证书**
- **允许多变量区域验证，只要最终输出的是严格、可复查的证书**



---

### 1.4 新系统的目标

相较于原系统，新系统希望做到：

1. **允许探索更广的势函数族**
   - 包括依赖 \($v$\) 的势函数
   - 包括依赖 \($uv\cap O_n$\) 长度的势函数
   - 包括低维非线性、分段、甚至有限拼接的势函数族

2. **允许验证多变量极值**
   - 不再强迫将所有问题压成单变量 \($g(\alpha)$\)
   - 允许在二维、三维甚至更高维紧区域上直接做严格上界验证

3. **允许极值出现在内部点**
   - 不再要求最大值必须落在端点或pivot
   - 允许出现 interior critical point，只要可以被严格控制

4. **仍然保持论文级正确性**
   - 每一步都对应明确的证明义务
   - 数值部分必须产出严格证书而不是仅做启发式采样

---

## 2 新的证明义务体系

原系统中使用的 N1–Q6 约束非常适合做第一次筛选，但其中 D4 和 Q5 绑定了 Xia 原证明的具体降维方式。新系统将它们改写为更一般的“证明义务”。

### 2.1 保留的硬约束

以下三类约束仍然作为不可放松的硬约束：

| 编号 | 名称 | 内容 | 原因 |
|------|------|------|------|
| S1 | 基础可行性 | \($\Phi\vert_{n=1}\le (\lambda-\pi/2)\|D\|$\)，理想为0 | 归纳起点必须成立 |
| S2 | 末端单调性 | \($\Phi_O\le \Phi_{O_{1,n-1}}$\) 或等价一盘扩展不增 | Prop 2 / 单步归纳必须依赖 |
| S3 | 分裂次可加性 | \($\Phi_O\le \Phi_{O_{1,j+1}}+\Phi_{O_{j+1,n}}$\) | Prop 3 / obstructed 分裂必须依赖 |
| S6 | 极端链下界证书 | Φ_{O*} ≥ −C'·\|P\|，C' 越小越好 | Lemma 3                          |

其中 S2 和 S3 与原系统中的 N2/N3 本质一致；S6 对应原 Q6，但形式上允许更一般的下界证书。

---

### 2.2 被宽松化的约束

#### S4：最坏点局部化证书（替代原 D4）

原 D4 要求：

- \($\Phi$\) 不依赖 \(v\)，或
- \($\Phi$\) 关于终点位置具有足够好的凸性

新系统改为：

> **S4：能够用某种严格证书，控制**
>$$
> \sup_{v\in A_b}\Upsilon_O(u,v)
> $$
> 
>**不要求极值必须位于端点或pivot点。**

这意味着以下任一种证书都可以接受：

- 端点/ pivot 证书
- 分段单调证书
- 凸/拟凸证书
- 驻点有限性 + 驻点区域上界证书
- 全区间严格包围证书

因此，势函数是否依赖 \(v\) 不再是先验淘汰条件。

#### S5：区域负性/下降证书（替代原 Q5）

原 Q5 的目标是保证某个变形量足够负，并最终压缩成四个单变量函数 $g_i(\alpha)\lt 0$。

新系统改为：

> **S5：在相关几何可行域 \(K\) 上，严格证明某个关键表达式**
>
> $$
> G_\theta(x)<0,\quad x\in K
> $$
>
> **其中 \(x\) 可以是多变量。**

也就是说，不要求一定压成单变量；可以直接对二维、三维或更高维区域上的函数做严格验证。

---

### 2.3 新的六个共享约束

供所有Agent统一使用的约束如下：

| 编号 | 性质 | 内容 |
|------|------|------|
| S1 | 基础可行性 | 单盘情形成立 |
| S2 | 末端单调性 | 加最后一个盘不使势函数上升 |
| S3 | 分裂次可加性 | 遇 obstruction 时势函数可拆分 |
| S4 | 最坏点局部化证书 | 不要求极值在端点，只要求极值可严格控制 |
| S5 | 区域负性/下降证书 | 不要求单变量，只要求在紧区域上可严格证伪为正 |
| S6 | 极端链下界证书 | 在 extremal chain 上势函数不能过于负 |

## 3 系统架构

### 3.1 三层结构概览

新系统仍采用三层架构：

1. **元控制层**：负责全局调度与历史管理
2. **自主探索层**：完成势函数设计、证明义务分解、局部证明与证书生成
3. **验证工具层**：提供符号、数值与严格证书支持

与原系统不同的是：自主探索层不再围绕“固定命题列表”运行，而是围绕**证明义务图（obligation graph）**运行。

```

            ▼                         │

│  势函数族生成器                                            │
│         │                                                  │
│         ▼                                                  │

│         │                                                  │
│         ▼                                                  │

│         ├────────→ 区域严格验证器（S4,S5）                 │
│         │                                                  │
```

### 3.2 元控制层

#### 3.2.1 Orchestrator（主脑）

**职责**：全局调度与路线切换，不直接参与具体证明。

**输入**：

- 通过报告
- 结构性失败报告
- 区域验证失败报告
- 反例点/反例盒
- 当前最优常数 \(C\) 与对应 \(\rho\)

**决策内容**：

- 若候选通过全部义务，则记录为当前可行势函数
- 若候选仅在区域局部失败，则要求在该局部邻域进行修正
- 若多个候选在同类区域反复失败，则判断是“证明模板不合适”还是“势函数族方向不对”
- 决定是否继续搜索、是否切换势函数族、是否转入证明整理阶段

**输出示例**：

- “继续搜索带 \(uv\cap O_n\) 线段长度项的候选，但优先维持 S2/S3 的线性可控性”
- “当前族在 \(\alpha\approx\pi/2\) 附近的拼接区域反复失败，建议引入平滑过渡项”
- “停止扩展新族，对当前最优候选进入正式证明编译阶段”

---

#### 3.2.2 Memory Manager（记忆管理员）

**职责**：维护结构化探索历史与反例模式库。

**维护内容**：

1. **候选势函数族树**
2. **每个候选的证明义务状态**
3. **失败区域与反例盒**
4. **哪些义务可沿用原证明**
5. **哪些义务必须改写**
6. **已知启发式经验**

**核心数据结构**：

```json
{
  "candidate_id": "Phi_family_07",
  "form": "a(r_n-r_1)-Σψ(H_i,V_i)+b·L(uv∩O_n)",
  "parameters": {"a": "...", "b": "..."},
  "obligation_status": {
    "S1": "pass",
    "S2": "pass",
    "S3": "pass",
    "S4": "pass_via_regional_certificate",
    "S5": "fail_in_box",
    "S6": "untested"
  },
  "counterexample_regions": [
    {
      "variables": ["α", "β", "γ"],
      "box": "[0.42,0.49]×[0.05,0.12]×[0.01,0.06]",
      "reason": "G_θ becomes positive"
    }
  ],
  "reusable_parts": ["Prop 2", "obstructed split skeleton"],
  "needs_redo": ["terminal localization", "transformation descent", "extreme-chain bound"],
  "pruned": false
}
```

## 4 自主探索层

### 4.1 势函数族生成器（Family Generator）

原系统中，设计师一次提出一个势函数。新系统改为：**一次提出一个势函数族**。

形式上，候选写成

$$
\Phi_\theta(O;u,v),
$$

其中 \(\theta\) 为有限维参数。

#### 允许的模板

1. **局部线性模板**
   $$
   a(r_n-r_1)-\sum_i \psi_\theta(H_i,V_i,\Delta r_i)
   $$

2. **带终点依赖模板**
   $$
   a(r_n-r_1)-\sum_i \psi_\theta(H_i,V_i)+b\cdot L(uv\cap O_n)
   $$

3. **分段模板**
   $$
   \psi_\theta=
   \begin{cases}
   \psi_\theta^{(1)}, & \alpha\le \pi/2\\
   \psi_\theta^{(2)}, & \alpha>\pi/2
   \end{cases}
   $$

4. **低维非线性模板**
   - 有理函数
   - 三角多项式
   - min/max 拼接
   - 光滑阈值项

#### 设计原则

- 尽量保留对 S2 / S3 友好的可分解结构
- 允许少量 \(v\)-dependence，但要可被 S4 处理
- 新项必须明确服务于某个难点，例如：
  - 改善小角区的下降量
  - 减少极端链下的负贡献
  - 修补 \(\alpha\approx\pi/2\) 附近的失衡

---

### 4.2 结构不变量检查器（Structural Invariant Checker）

这是第一道筛选器，只负责 S1/S2/S3 的快速检查。

#### 其功能是：

- 检查单盘情形是否满足基础可行性
- 检查删除末端盘时势函数差是否可控
- 检查 obstructed split 后是否可写成次可加形式
- 判断某个候选是否具有继续深入分析的必要

#### 与原系统的差异

原系统中，若候选引入 \(v\)-dependence，往往会被 D4 预期直接判定“高风险”。新系统中，这**不再构成直接剪枝理由**。只要 S1/S2/S3 通过，即可进入下一层。

---

### 4.3 最坏情形分析器（Worst-case Geometry Analyzer）

这是新系统中最关键的分析模块。

其任务不是强行把问题压成单变量，而是：

1. 给出变量参数化
2. 推导几何可行域 \(K\)
3. 尝试寻找可用的局部化证书
4. 若降维失败，则生成区域验证任务

#### 输出形式

```yaml
worst_case_plan:
  variables: [α, β, γ, t]
  feasible_region:
    - 0 < α < π
    - -sin α <= β <= sin α
    - 0 <= γ <= π/2 - (α-β)/2
    - t in [0,1]
  localization_attempts:
    - endpoint certificate: fail
    - pivotal certificate: inconclusive
    - piecewise monotonicity: partial
    - interior critical-point search: needed
  next_step:
    - send full region to certifier
```

#### 三种典型输出路径

**路径 A：成功降维**
- 证明最大值出现在边界
- 继续交给 1D/2D 验证器

**路径 B：分段降维**
- 将区域分成几个子区域
- 每一块分别选择最合适的证书

**路径 C：完全区域验证**
- 无法进一步降维
- 直接交给多变量严格验证器

---

### 4.4 区域严格验证器（Region Certifier）

这是替代原“单变量 \(g_i(\alpha)\lt0\) 验证器”的核心模块。

#### 输入

- 目标表达式 \(F(x)\)
- 变量区域 \(K\subset \mathbb{R}^d\)
- 验证目标：\(\sup_{x\in K}F(x)<0\)

#### 输出

- “通过”及其严格证书
- 或者“失败”及反例盒

#### 两层模式

##### （1）搜索模式（非严格）
用于快速找问题：

- 随机采样
- 网格扫描
- 局部优化
- 自动寻找疑似最坏点

##### （2）证明模式（严格）
用于真正出证书：

- 区间算术
- 自动微分界
- 多变量 Lipschitz 上界
- Branch-and-Bound
- Bernstein / Chebyshev 包络
- McCormick relaxation
- 必要时调用 SOS/SDP

#### 证书对象示例

```json
{
  "obligation": "S5",
  "function": "G_theta(α,β,γ)",
  "region": "K",
  "proof_type": "interval_branch_bound",
  "subboxes": [
    {"box": "[0,0.5]x[-0.4,0.4]x[0,0.2]", "upper_bound": -0.013},
    {"box": "[0.5,1.0]x[-0.8,0.8]x[0.1,0.4]", "upper_bound": -0.006}
  ],
  "global_upper_bound": -0.006
}
```

---

### 4.5 极端链下界器（Extreme-chain Bounder）

原证明中的 Lemma 3 最终依赖 aligned chain、heavy/light path 抵消以及 Cauchy–Schwarz。新系统中，由于势函数可能更复杂，这个模块需要升级。

#### 职责

- 对给定势函数族，尝试构造
  $$
  \Phi_{O^\*}\ge -C\cdot |P_{O^\*}|
  $$
  的证书
- 若势函数中新增了新统计量 \(W_i\)，则自动寻找
  $$
  \sum_i(aH_i+bV_i+cW_i)\le C|P|
  $$
  之类的线性或凸组合上界

#### 可用方法

- 对齐变换
- 路径抵消
- 向量范数上界
- Cauchy–Schwarz / Hölder
- 小规模 LP / convex relaxation

#### 输出

- 最终可用的 \(C\)
- 推导链条
- 哪些部分延续原 Lemma 3
- 哪些部分为新增估计

---

### 4.6 反例驱动修正器（Counterexample-Guided Refiner）

这个模块负责把验证失败转成“可操作的改进建议”。

#### 输入

- 区域验证失败盒
- 结构不变量失败原因
- 极端链下界失败样本

#### 输出类型

1. **结构性失败**
   - “平方项导致 S3 根本不可能成立”
   - “新增终点项破坏 S2，且不可通过系数修复”

2. **局部区域失败**
   - “小 \(\gamma\) 且中等 \(\beta\) 时导数翻正”
   - “\(\alpha=\pi/2\) 附近分段拼接不光滑”

3. **极端链失败**
   - “新增项在 aligned reduction 后无法被 \(|P|\) 吸收”
   - “heavy/light 抵消不再有效”

这样，失败不再只是“该候选被剪枝”，而是能反馈到下一轮候选设计中。

---

### 4.7 证明编译器（Proof Compiler）

当某个候选通过全部义务后，系统会将离散证书、局部分析和全局推理整合为一份可写入论文/附录的证明草稿。

#### 输出结构

- 哪些命题完全复用原论文
- 哪些命题仅改写参数
- 哪些命题改为区域证书
- 数值附录应包含哪些严格验证日志
- 最终得到的常数 \(C\)、\(\lambda\)、\(\rho\)

#### 目标

不是只输出“搜索成功”，而是输出：

> 一条完整、分层、可审阅、可复现实验的证明骨架。

---

## 5 验证工具层

### 5.1 区间算术与 Branch-and-Bound

用于做区域上的严格上界验证，是 S4/S5 的基础工具。

**适用场景**：

- 多变量函数极值验证
- 内点临界点附近的严格包围
- 边界与内部的统一处理

---

### 5.2 Lipschitz 优化器

保留原系统中单变量 Piyavskii 的思想，但推广到低维区域。

**适用场景**：

- 函数有明确 Lipschitz 上界
- 希望快速给出全局上界
- 在 2D/3D 紧区域上做保守验证

---

### 5.3 符号计算器

**实现建议**：SymPy + 自动微分

**功能**：

- 求偏导
- 化简代数式
- 推导区域边界
- 生成局部 Taylor 上界
- 为数值验证器提供导数界

---

### 5.4 几何采样器

**作用**：只负责启发式找难点，不负责最终证明。

它可以：

- 自动搜索最坏参数区域
- 识别可能的 interior critical points
- 给区域严格验证器做 warm start

---

### 5.5 SOS / SDP 接口（可选）

当候选势函数经过代数化或三角替换后可变成半代数问题时，可以尝试：

- Sum-of-Squares
- 半定规划
- 多项式上界证书

这不是默认工具，但对某些低维非线性模板可能非常有效。

---

### 5.6 领域文献RAG

继续保留，但用途略有升级：

1. 寻找新的势函数灵感
2. 寻找极端链下界技巧
3. 借鉴全局优化与几何不等式证法
4. 对齐 Xia 原证明中可复用的技术组件

## 6 工作流程

### 6.1 正常流程（候选通过）

```
Orchestrator → 势函数族生成器:
  "探索带终点依赖项的候选，优先考虑 uv∩O_n 长度"

势函数族生成器:
  提出 Φ_θ = a(r_n-r_1)-Σψ(H_i,V_i)+b·L(uv∩O_n)

→ 结构不变量检查器:
  S1 通过；S2 通过；S3 通过

→ 最坏情形分析器:
  原端点/ pivot 证书失效，因为 Φ 依赖 v
  生成三维可行域 K(α,β,γ)

→ 区域严格验证器:
  对 S4 做全弧上界证书
  对 S5 做 ∂Υ/∂X_{o_n} 的区域负性证书
  全部通过

→ 极端链下界器:
  证明新增项在 aligned chain 上满足
  Φ_{O*} ≥ -C·|P|

→ 证明编译器:
  输出完整证明骨架与验证附录

→ 正确性检查员:
  通过

→ Orchestrator:
  记录为当前最优候选
```

---

### 6.2 正常流程（局部区域失败）

```
势函数族生成器 → 候选 Φ_θ

结构不变量检查器:
  S1/S2/S3 全过

最坏情形分析器:
  无法降维，生成 K(α,β,γ)

区域严格验证器:
  在 box B=[0.42,0.49]×[0.05,0.12]×[0.01,0.06] 中发现上界 > 0

→ 反例驱动修正器:
  判断失败集中在小 γ、适中 β 区域
  建议增强该区域的下降项，或减弱终点依赖项

→ Orchestrator:
  保留该族，不直接剪枝
  指令：针对该 box 邻域调参数重试
```

---

### 6.3 正常流程（结构性失败）

```
势函数族生成器:
  提出包含 Σ(Δr_i)^2 的候选

结构不变量检查器:
  S3 失败
  原因：平方项不满足 split 次可加性

→ 反例驱动修正器:
  判定为“结构性不可修复失败”

→ Orchestrator:
  剪枝该族
  记录启发式：避免凸型非可加累积项
```

---

### 6.4 正常流程（极端链下界失败）

```
候选通过 S1–S5

→ 极端链下界器:
  发现新增统计量 W_i 在 aligned reduction 后无法吸收到 |P| 中
  无法得到 Φ_{O*} ≥ -C·|P|

→ 反例驱动修正器:
  建议将 W_i 改成可沿 path 抵消的形式

→ Orchestrator:
  保留主结构，回到生成器做轻量修补
```

## 7 设计决策总结

| 决策 | 选择 | 理由 |
|------|------|------|
| 是否继续保留三层架构 | **是** | 原系统的分层已经合理，只需升级探索层 |
| 候选单位 | **势函数族而非单个势函数** | 便于参数搜索与局部修补 |
| D4 的处理 | **改为 S4：最坏点局部化证书** | 不再先验要求极值在端点或pivot |
| Q5 的处理 | **改为 S5：区域负性证书** | 支持多变量严格验证 |
| 数值验证策略 | **搜索模式 + 严格证明模式** | 兼顾效率与可信度 |
| 失败反馈机制 | **反例盒回流而不是简单剪枝** | 便于候选迭代 |
| Q6 的处理 | **升级为一般极端链下界器** | 兼容更复杂势函数项 |
| 证明产物 | **证书+证明骨架并行输出** | 便于后续写论文与复查 |

## 8 与原系统的关键差异

### 原系统的核心假设

原系统默认：

- 好势函数大概率不依赖 \(v\)
- 最坏终点可以压到端点或 pivot
- 难点最终可以压缩成单变量函数验证
- 因而探索重点是“找一个符合原证明模板的势函数”

### 新系统的核心假设

新系统认为：

- 好势函数未必不依赖 \(v\)
- 最坏点未必在端点，可能在 interior
- 多变量函数完全可以直接严格验证
- 因而探索重点是“找一个能完成全部证明义务的势函数族”

一句话概括：

> 原系统是“固定证明模板 + 搜势函数”；
>  
> 新系统是“固定证明义务 + 搜证书类型与势函数族”。
